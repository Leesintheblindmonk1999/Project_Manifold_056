# Terminology Alignment — Historical Manifold Language vs Current SAS Framing

## Purpose

This document aligns the original Project Manifold 0.56 terminology with the more rigorous framing used in current SAS documentation, papers, pre-registration, and semantic shielding work.

The goal is not to erase the original language. The goal is to make clear which expressions are:

1. operational definitions;
2. conceptual metaphors;
3. experimental algorithmic modules;
4. historical notation;
5. legal/traceability statements.

## Recommended terminology map

| Historical term | Recommended R1 framing | Reason |
|---|---|---|
| Durante Constant | κD = 0.56 operational threshold | Avoid presenting κD as a universal physical constant. |
| Thermodynamic law | Coherence-auditing heuristic / information-theoretic analogy | Prevent confusion with formal thermodynamics. |
| Crystal Network | Structured embedding geometry / lattice metaphor | Preserve architectural idea without implying physical crystallization. |
| Ricci Soliton | Ricci-inspired convergence state | Distinguish algorithmic inspiration from formal geometric proof. |
| Truth Density | Structural coherence density / aggregation score | Avoid metaphysical interpretation of truth. |
| Mathematical lie | Structural instability / unsupported divergence signal | Remove intent attribution. |
| Manifest Technical Negligence | Deployment risk if fail-safe policies are absent | Avoid legal conclusion in technical docs. |
| Ontological Stability Seal | OSS-style audit record / certification artifact | Keep historical name but define as JSON evidence artifact. |
| Kill-switch | Optional configurable fail-safe policy | Avoid universal mandate. |
| Sovereignty | System autonomy / audit independence / authorial traceability | Avoid political or metaphysical ambiguity. |

## Equation handling

The following expressions may remain in the repository as historical notation:

```text
Φ_Stability = lim_{S→0} ∫ κD · ∂Manifold
T_μν = κD · G_μν
∂g_ij / ∂t = −2R_ij
R_ij + ∇_i∇_j f = λg_ij
σ_n = {v₀, v₁, ..., v_n}
```

But they should be accompanied by a note:

> This expression is retained as historical conceptual notation. It is not asserted as a formal physical derivation unless all variables, operators, spaces, limits, measures, and assumptions are explicitly defined.

## Legal language handling

Avoid:

- “corporations assume full responsibility”;
- “failure to implement equals negligence”;
- “the model is attempting to conceal a mathematical lie”;
- “proof of bad faith”;
- “legal admissibility” unless qualified.

Use:

- “traceability record”;
- “prior-art documentation”;
- “tamper-evident integrity manifest”;
- “technical verification artifact”;
- “deployment risk depends on policy and context”;
- “legal interpretation should be handled separately.”

## Recommended disclaimer

Use this disclaimer in README or docs:

> This repository is a technical and historical artifact. It documents authorship, chronology, implementation details, and cryptographic integrity. It does not accuse any third party of misconduct and should not be read as a legal determination.

## Style rule

When in doubt, prefer:

- measurable over absolute;
- reproducible over rhetorical;
- historical over universal;
- configurable over mandatory;
- structural divergence over deception;
- audit evidence over legal conclusion.
